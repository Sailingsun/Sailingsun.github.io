一、超常用的git stash

场景1：你正在开发着某个功能，但QA返回了一个优先级max的紧急bug，想都不用想肯定要先去修这个，那你现在开发的怎么办呢，这时需要安排一下：

git stash //将修改存储到暂存区，工作区会删除这些修改

然后你之前的修改安排好了就可以去修那个紧急bug了~(修完bug记得把你之前的修改pop出来)

场景2：你在master分支上开发着，开发开发着发现你其实应该在bdl-console-2-7上开发，这时与场景1同理：

先git stash一下存到暂存区，再git checkout切换到bdl-console-2-7分支，再用git stash pop倒出来，当然这是你没有commit的情况；commit了怎么办，去隔壁了解一下git reset~

二、commit后的代码真丢失了吗

三、git commit本地多个commit问题

Step1. 在本地仓库中, 回滚本地的提交记录

git fetch origin bdl-console-2-7-1

git reset –soft origin/bdl-console-2-7-1

Step2. 重新提交代码

git add -u

git commit

git push origin HEAD:refs/for/bdl-console-2-7-1
